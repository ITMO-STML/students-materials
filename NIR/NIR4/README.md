# LEMON Demo Interface

**LEMON** — *Linguistically Enriched Measure of Ontological Normalization*. Метод оценивает семантическую полноту графа знаний через сравнение канонических фактов исходного текста и фактов, восстановленных из структуры графа.

## 1. Что делает программа

Интерфейс выполняет полный демонстрационный сценарий:

1. принимает на вход статью в формате `.pdf` или `.txt`;
2. извлекает текст из PDF, при необходимости может использовать OCR;
3. очищает текст и выделяет содержательную часть статьи;
4. строит граф знаний на основе section-aware proposition extraction;
5. нормализует извлечённые факты в canonical facts;
6. рассчитывает метрику LEMON, precision, F1 и MINE-1 compatible score;
7. показывает граф визуально прямо в браузере;
8. сохраняет результаты в JSON и Cypher;
9. при наличии настроенного Neo4j импортирует граф в базу и выводит ссылку на Neo4j Browser.

## 3. Требования

Рекомендуемая версия Python: **Python 3.11**.

Основные библиотеки:

- `streamlit` — веб-интерфейс;
- `PyMuPDF` и `pypdf` — чтение PDF;
- `pytesseract` и `Pillow` — OCR для PDF-сканов;
- `networkx` — внутренняя работа с графами;
- `plotly` — интерактивная визуализация графа;
- `rapidfuzz`, `nltk`, `scikit-learn` — нормализация и сопоставление признаков;
- `neo4j` — необязательный экспорт в Neo4j.

Для OCR требуется установленный системный пакет **Tesseract OCR**. Если используются только PDF с текстовым слоем, Tesseract не обязателен.

## 4. Установка

Распакуйте архив и перейдите в папку проекта:

```bash
cd semgraph_kg_project
```

Создайте виртуальное окружение:

```bash
python3.11 -m venv .venv
```

Активируйте его:

```bash
# Linux / macOS
source .venv/bin/activate

# Windows PowerShell
.venv\Scripts\Activate.ps1
```

Установите зависимости:

```bash
pip install --upgrade pip
pip install -r requirements.txt
pip install -e .
```

## 5. Запуск интерфейса

Запуск Streamlit-приложения:

```bash
streamlit run app.py
```

После запуска откройте адрес, который появится в терминале, обычно:

```text
http://localhost:8501
```

В интерфейсе необходимо загрузить статью в формате PDF или TXT и нажать кнопку **«Построить граф и рассчитать метрики»**.

## 6. Используемые исходные данные

При демонстрации на защите в качестве входных данных используется научная статья в формате PDF или TXT. В экспериментах проекта рассматривался корпус из 200 научных статей: 100 русскоязычных и 100 англоязычных. В интерфейс можно загрузить как одну статью из этого корпуса, так и новый документ.

Для входного документа система ожидает один из вариантов:

- PDF с текстовым слоем;
- PDF-скан, если установлен Tesseract OCR;
- текстовый файл UTF-8.

Исходные данные после загрузки не изменяются. Временная копия файла создаётся только на время выполнения расчёта.

## 7. Результаты работы программы

После обработки интерфейс показывает:

- количество узлов графа;
- количество рёбер графа;
- значение LEMON;
- precision;
- F1;
- значение MINE-1;
- интерактивную визуализацию графа;
- JSON-представление графа;
- подробные результаты метрик;
- Cypher-файл для импорта в Neo4j.

Доступны кнопки выгрузки:

- `graph.json` — полный граф знаний;
- `metric.json` — результат LEMON;
- `mine1.json` — результат MINE-1;
- `summary.json` — краткая сводка;
- `graph_import.cypher` — команды для Neo4j.

## 9. Консольный запуск без интерфейса

Для пакетной обработки PDF можно использовать консольную команду:

```bash
semgraph evaluate-pdf --pdf path/to/article.pdf --out-dir outputs/article_demo
```

Для TXT-файла:

```bash
semgraph evaluate-text --text-file path/to/article.txt --out-dir outputs/article_demo
```

Результаты будут сохранены в указанной папке.

## 10. Проверка тестов

Для запуска тестов:

```bash
pytest -q
```

Тесты проверяют базовые модули чтения PDF, извлечения графа, расчёта LEMON, расчёта MINE-1 и вспомогательные операции.

## 11. Методическая схема эксперимента

Экспериментальный запуск строится по следующему сценарию:

```text
статья → предобработка → извлечение графа → canonical facts → LEMON → MINE-1 → отчёт
```

Метрика LEMON рассчитывается как взвешенная полнота канонических фактов:

```text
LEMON = sum(w(f) * m(f)) / sum(w(f))
```

где `w(f)` — вес факта, а `m(f)` — степень покрытия факта в графе. Пропущенные факты уменьшают итоговую оценку через нулевой вклад, частично сохранённые факты дают неполный вклад, а избыточные факты отражаются через precision и F1.
