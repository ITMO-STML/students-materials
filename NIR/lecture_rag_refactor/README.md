# Lecture RAG (2-Agent) — генерация лекций из документов

Прототип Retrieval-Augmented Generation (RAG) системы для автоматической генерации лекций по заданной теме на основе пользовательских материалов (PDF/DOCX/PPTX/TXT/MD).  
Архитектура: **централизованный оркестратор + 2 LLM-агента (Lector → Reviewer/Enricher) + RAG (FAISS retriever)**.

## Описание работы системы

1) **Загрузка материалов**  
Документы из папки `lecture_rag/resources/` загружаются через адаптер `adapters/loaders_langchain.py` (поддержка нескольких форматов).

2) **Индексация (RAG)**  
Тексты разбиваются на чанки (`rag/splitters.py`), для чанков строятся эмбеддинги, затем создаётся векторный индекс **FAISS** и `retriever` (`rag/index.py`).

3) **Генерация черновика (Agent: Lector)**  
Оркестратор передаёт **тему + найденные релевантные чанки** в цепочку LangChain (`rag/chains.py`), которую использует агент `agents/lector.py`. На выходе — черновик лекции.

4) **Улучшение результата (Agent: Reviewer/Enricher)**  
Черновик передаётся агенту `agents/enricher.py`, который улучшает структуру/качество текста (проверка, дополнение, исправление формулировок). На выходе — финальная версия.

5) **Результат**  
Система возвращает финальный текст лекции (и при необходимости черновик).

Он содержит:
* установку зависимостей
* настройку API_KEY / BASE_URL
* загрузку документов в lecture_rag/resources
* запуск CLI (и при необходимости — Streamlit через туннель)
