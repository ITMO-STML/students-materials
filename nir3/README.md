Правилоориентированная морфологическая дизамбигуация с использованием CG3 и MDL-оценки

1. Назначение
Данный программный комплекс предназначен для экспериментального анализа морфологической неоднозначности и её разрешения с использованием:
морфологического анализатора uniparser,
контекстной дизамбигуации на основе CG3,
набора явных лингвистических правил,
и оценки их вклада с помощью MDL-подобной метрики.
Система ориентирована на исследовательские задачи и позволяет сравнивать различные стратегии выбора морфологического анализа (top-1) при сохранении полного набора альтернатив.

2. Общая схема работы
Общая схема работы программного комплекса включает следующие этапы:
Морфологический анализ каждого токена (uniparser).
Формирование списка кандидатов (top-k).
Выбор базового top-1 анализа (baseline).
(Опционально) контекстная дизамбигуация с помощью CG3.
(Опционально) применение лингвистических правил.
Фиксация типов неоднозначности в виде диагностических исключений.
(Опционально) привязка семантического значения (word sense) через RuWordNet.
Сравнение результатов различных режимов.
Важно, что система не удаляет альтернативные анализы, а только принимает решение о выборе top-1, сохраняя информацию о наличии и типах неоднозначности.

3. Реализованные режимы работы
В системе реализовано четыре сопоставимых режима, используемых для экспериментального анализа:
A_topk - морфологический анализ без CG3 и без правил (baseline). Выбор первого кандидата, возвращённого анализатором,
B_rules - применение лингвистических правил поверх baseline без CG3,
C_cg3 - контекстная дизамбигуация с использованием CG3 без правил,
D_cg3rules - совмещение CG3 и лингвистических правил.
Такая организация позволяет отдельно оценить вклад:
внутренних приоритетов анализатора,
правил,
CG3,
и их комбинации.

4. Диагностика неоднозначности
Система различает решение и наличие неоднозначности.
Для каждого токена автоматически фиксируются следующие типы диагностических исключений:
AmbiguityException — наличие более одного морфологического кандидата,
SegmentationConflictException — различие морфемной сегментации (wfGlossed),
TagConflictException — различие морфологических тегов / gloss,
ResolvedByRule — случай, когда правило реально изменило выбранный top-1 анализ.
Таким образом, даже после выбора top-1 система сохраняет информацию о структуре неоднозначности, что важно для последующего анализа.

5. Семантический слой (опционально)
При включении семантического режима система:
извлекает русский перевод (ru_hint) из выбранного морфологического анализа;
осуществляет поиск соответствующих синсетов в RuWordNet;
фиксирует факт наличия или отсутствия привязки word sense.
Семантический слой используется исключительно как диагностический и не влияет на морфологический выбор.

6. Запуск и использование
6.1 Системные требования
Python ≥ 3.10
ОС: macOS / Linux / Windows
Установленные внешние зависимости:
uniparser-udmurt
streamlit
(опционально) ruwordnet
CG3 / VISLCG3 (при использовании контекстной дизамбигуации)
6.2 Установка и запуск
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app_streamlit.py
После запуска интерфейс доступен в браузере по адресу
http://localhost:8501.

8. Интерфейс пользователя
Интерфейс, реализованный с использованием Streamlit, позволяет:
вводить отдельные предложения или токены,
запускать анализ во всех четырёх режимах,
просматривать
выбранный top-1 анализ,
список альтернативных кандидатов,
диагностические исключения,
информацию о применённых правилах,
(опционально) статус семантической привязки.
Таким образом, входными данными является текстовое предложение, а выходными — структурированное представление морфологического анализа и принятых решений.

8. Результаты и их интерпретация
Результатом работы программы является не только выбранный морфологический анализ, но и:
информация о наличии и типе неоднозначности,
факт применения или неприменения правил,
различия между режимами baseline, CG3 и rule-based анализа.
Это позволяет использовать систему как инструмент экспериментального сравнения стратегий дизамбигуации, а не как “чёрный ящик”.

9. Назначение репозитория
Репозиторий предназначен для:
воспроизводимости экспериментов,
анализа вклада правил и CG3,
демонстрации интерпретируемого подхода к морфологической дизамбигуации.
