## Содержание
### Подготовленные модели
В данном ноутбуке демонстирируется две подготовленные модели.

Первая – это бейзлайн, в котором не применяется адапатация самообученной нейронной сети. Id модели 059259c395e34f71a88cc6a0925953a1.

Вторая модель – лучший чекпоинт после выполнения адаптации. Id модели 8a8bbaa9405e4c33a450d85b960f5965.

### Тестовый прогон моделей (инференс на подготовленном датасете)
Прогон моделей осуществляется на GPU сервере с использованием системы логирования Clearml.

Общий размер тестового датасета равен 868301 примеров. Инференс занимает 2:27:29 времени.

По результатам прогона каждой модели формируется файл score.tsv, в котором сохранены пары значений: путь в аудио файлу и вероятность спуфинга.

### Метрика оценки качества
В качестве сравнительной метрики качества используется Equal-Error-Rate (EER), реализована в функции calc_eer.

На вход данная функция принимает numpy массив подсчитанных scores и бинарные метки targets.

### Построение метрик
После этапа инференса объединяется файл с разметкой и полученными результатами по ключу путь файла (wav_path). 

После этого на строится общий EER по колонкам scores и target. А также EER по различным датасетам, используя фильтр pandas датафрейма по имени датасета.
