# NIR: Статус проекта — передача в новый диалог

## Тема НИР
Генерация обучающих данных с варьированием сложности для моделей семантического поиска.

## Что сделано (Фаза 0 — Sanity Check)

### Инфраструктура
- Проект `NIR_2` на `W:\Jupyter\NIR_2\`
- LLM: Qwen2.5-7B через **Ollama** на GPU (RTX 4060 Laptop 8GB)
- Кириллица в имени пользователя Windows → Ollama запускаем вручную из `start_env.bat`
- 15 рукотворных sanity-примеров (фактоидные пары q/d⁺ на русском)

### Результаты Фазы 0 (2 прогона, второй — с обновлёнными промптами)

**Fact extraction — работает:**
- JSON parse: 100%
- 2.1 факта/документ (хотим 3-5, but usable)
- Hallucination 8.8% (часть — морфологические варианты, не истинные галлюцинации)
- Recall 62% vs ожидаемых фактов (частично занижен из-за русской морфологии)

**Counterfactual generation — работает отлично:**
- 0% self-copy, 1.00 length ratio, стиль сохраняется
- Мутации осмысленные (Толстой→Пушкин, 1380→1385, etc.)
- 9.7% fact-still-present (приемлемо)

**LLM-judge — слабое место:**
- 100%/100% на positive/negative контролях (Run 1)
- 48% accuracy на контрфактах (Run 1) → 61% (Run 2)
- Run 2 ввёл регрессию: 86.7% на позитивах (judge стал слишком строгим)
- **Паттерн:** judge плохо ловит числовые/датовые изменения
- **Решение:** chain-of-thought judge + опора на cross-encoder (DualJudge)

### Ключевые архитектурные решения
1. **δ считается ансамблем энкодеров** (E5 + BGE-M3), а не одним
2. **Open-ended факт-типы** (не зашитая таксономия)
3. **Двойная верификация** (CE + LLM) — CE компенсирует слабости LLM-judge
4. **Стратификация по δ** — 3 бакета (easy/medium/hard) вместо одного диапазона
5. **Gradient positives — дерево**, не цепочка

### Базовая модель для дообучения
- ai-forever/ru-en-RoSBERTa (Russian-specialized)

### Данные
- Разработка: RuBQ 2.0 (~2900 пар)
- Скейл: mMARCO-ru
- Eval: MIRACL-ru, RuBQ-test
