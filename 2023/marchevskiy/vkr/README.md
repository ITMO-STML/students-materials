## Содержание
### Описание подходов
Большинство алгоритмов автоматического распознавания речи способны осуществлять транскрибирование одного определенного языка. Это связано со сложностями, возникающими в процессе декодирования нескольких разных языков в рамках одного алгоритма одновременно, а также ограниченностью объемов хорошо размеченных мультиязычных речевых баз. Для возможности работы с речевыми сигналами с переключением языка используются модели языковой детекции и диаризации. Алгоритм использования языковой детекции и диаризации с моноязычными системами распознавания речи (ASR) в целях транскрибирования мультиязычной речи можно описать следующим образом:
- Сегментация аудиопотока: на первом этапе входной аудиосигнал разделяется на сегменты. Это может осуществляться с использованием алгоритмов голосовой активности (VAD, Voice Activity Detection), которые отделяют фрагменты, содержащие речь, от тишины или фона. Такой подход позволяет уменьшить объем данных для последующей обработки.
- Языковая детекция для каждого сегмента: на этом шаге производится определение языка каждого сегмента речи. Для этого используются языковые детекторы (Language Identification Systems). Они работают путем анализа акустических особенностей речи или фонетических паттернов, чтобы классифицировать каждый сегмент как принадлежащий определенному языку из заданного набора. Важно учитывать, что точность языковой детекции может зависеть от длины сегмента: чем больше доступного контекста, тем выше вероятность корректного вывода.
- Диаризация (опционально): чтобы обойти ограничения работы языкового детектора, связанные с длинами сегментов, можно использовать алгоритм языковой диаризации. Такой подход позволяет лучше обрабатывать короткие вхождения одного языка в другой (например, одно или несколько слов).
- Маршрутизация сегментов в соответствующие ASR модели: после определения языка для каждого сегмента, он направляется в соответствующую моноязычную систему автоматического распознавания речи (ASR), предназначенную для обработки определенного языка. Например, сегменты на английском языке отправляются в англоязычную модель ASR, сегменты на французском — во франкоязычную и так далее.
- Транскрибирование сегментов: каждая модель ASR выполняет транскрипцию для поступивших на вход сегментов на заданном языке. На данном этапе получается текстовая интерпретация речи для каждого фрагмента, соответствующая языку модели, используемой для обработки.
- Сбор результатов: результаты транскрипции всех сегментов объединяются в единый текстовый выход. На этом этапе может также сохраняться информация о границах сегментов, временные метки, говорящие (из диаризации) и язык каждого сегмента. Это позволяет получить многоуровневую аннотацию транскрипта.

В демонстрационном ноутбуке отображены разработанные алгоритмы:
- обучение модели языковой идентификации;
- языковая диаризация на основе кластеризации высокоуровневых представлений, полученных алгоритмом языковой идентификации как экстрактором признаков.

### Тестовые данные
В качестве тестовых данных для оценки качества модели языковой идентификации использованы следующие базы данных:
- FLEURS. представляет собой n-сторонний параллельный набор речевых данных на 102 языках, построенный поверх бенчмарка машинного перевода FLoRes-101, в среднем 12 часов речи на каждом языке.
- Minds-14. Набор данных созданный в результате использования коммерческого голосового помощника и реальных потребностей отрасли: он охватывает 14 сценариев в банковской сфере на 14 различных языковых вариантах, что делает его наиболее полным на сегодняшний день многоязычным набором данных для обнаружения сценариев.
